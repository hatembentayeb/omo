AWS Costs Plugin

Monitor AWS spending with Cost Explorer, budgets, and forecasts. View daily and monthly cost breakdowns, track budget alerts, forecast future spending.

aws finops cloud

Profiles are loaded from two sources: the config file (with optional KeePass) and ~/.aws/credentials.

Prerequisites

  • AWS account with Cost Explorer enabled
  • IAM permissions: ce:GetCostAndUsage, ce:GetCostForecast, budgets:ViewBudget
  • AWS credentials (via ~/.aws/credentials, environment variables, or config)

Configuration

Config: ~/.omo/configs/awsCosts/awsCosts.yaml

This file only controls which environments are enabled and UI settings. All AWS credentials are stored in KeePass.

~/.omo/configs/awsCosts/awsCosts.yaml
environments:
  - name: default
    enabled: true
  - name: production
    enabled: true

ui:
  refresh_interval: 300
  default_time_range: "LAST_30_DAYS"
  default_granularity: "DAILY"
  default_region: "us-east-1"
  enable_budgets: true
  enable_forecasts: true
KeePass-first

All AWS credentials (access key, secret key, region, role ARN) are stored exclusively in KeePass entries under awsCosts/<environment>/<name>. The YAML config only controls which environments are enabled and UI behavior. Profiles from ~/.aws/credentials are also merged automatically.

KeePass Secret Schema

Path: awsCosts/<environment>/<name>

Example: awsCosts/production/billing

Troubleshooting

  • No cost data — Ensure Cost Explorer is enabled in the AWS Console.
  • Access denied — Verify IAM permissions for Cost Explorer and Budgets.
  • Wrong region — Cost Explorer is a global service; set region to us-east-1.
  • Stale data — Increase refresh_interval or press R to refresh manually.
