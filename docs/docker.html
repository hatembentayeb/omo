Docker Plugin

Manage Docker containers, images, networks, and volumes. Start, stop, restart, logs, exec, prune, and compose support.

containers devops infrastructure

Prerequisites

  • Docker daemon running (local or remote)
  • User in docker group or sudo access
  • For remote: TCP access to Docker API port

Configuration

Config file: ~/.omo/configs/docker/docker.yaml

This file only controls which environments are enabled and UI settings. All connection details are stored in KeePass.

~/.omo/configs/docker/docker.yaml
environments:
  - name: development
    enabled: true
  - name: production
    enabled: true
  - name: staging
    enabled: true

ui:
  refresh_interval: 5
  max_containers_display: 100
  max_images_display: 100
  show_all_containers: true
  log_tail_lines: 500
  enable_stats: true
  enable_compose: true
KeePass-first

All connection details (Docker host URL, TLS certificate path) are stored exclusively in KeePass entries under docker/<environment>/<name>. The YAML config only controls which environments are enabled and UI behavior.

KeePass Secret Schema

Path convention: docker/<environment>/<name>

Example: docker/production/remote-server

omo (root)

Views

The Docker plugin provides these views:

  • Containers (default) — List and manage containers
  • Images — Browse and manage images
  • Networks — View and configure networks
  • Volumes — Manage volumes
  • Stats — Container resource usage
  • Inspect — Detailed container/image inspection
  • Logs — View container logs
  • Compose — Docker Compose projects
  • System — Docker system information

Troubleshooting

  • Permission denied — Add user to docker group
  • Cannot connect to daemon — Ensure Docker is running
  • TLS errors — Verify cert_path and certificate files
