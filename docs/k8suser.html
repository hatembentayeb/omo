K8s User Plugin

Manage Kubernetes user certificates with certificate-based authentication. Generate CSRs, create RBAC bindings, store credentials securely in KeePass.

kubernetes security rbac

Clusters are loaded from two sources: config file (with optional KeePass) AND host ~/.kube/config (kubectl contexts).

Prerequisites

  • Kubernetes cluster (1.19+)
  • OpenSSL installed (for certificate generation)
  • kubeconfig or bearer token for cluster access
  • Cluster admin permissions for CSR approval and RBAC

Configuration

Config: ~/.omo/configs/k8suser/k8suser.yaml

This file only controls which environments are enabled and UI settings. All cluster connection details are stored in KeePass.

~/.omo/configs/k8suser/k8suser.yaml
environments:
  - name: development
    enabled: true
  - name: production
    enabled: true

ui:
  refresh_interval: 30
  default_namespace: "default"
  cert_validity_days: 365
  enable_role_manager: true
  enable_access_testing: true
  enable_export: true
KeePass-first

All cluster connection details (server URL, bearer token, kubeconfig path, context, CA cert) are stored exclusively in KeePass entries under k8suser/<environment>/<name>. The YAML config only controls which environments are enabled and UI behavior. Clusters from ~/.kube/config are also merged automatically.

KeePass Secrets

Path: k8suser/<environment>/<name>, e.g. k8suser/production/main-cluster

Example KeePass structure
omo (root)
  └── k8suser
       ├── development
       │    └── minikube     (kubeconfig=~/.kube/config, context=minikube)
       └── production
            └── main-cluster (URL=https://k8s.prod.internal:6443, Password=eyJhbG..., ca_cert=/etc/k8s/ca.pem)
OpenSSL required

OpenSSL is required for certificate generation. Install it via your package manager: sudo pacman -S openssl or sudo apt install openssl.

Troubleshooting

  • OpenSSL not found — Install the openssl package via your package manager.
  • Cannot connect to cluster — Verify server URL, token, or kubeconfig configuration.
  • CSR rejected — Ensure you have cluster admin permissions.
  • Certificate expired — Check the cert_validity_days setting in the config.
