ArgoCD Plugin

Manage ArgoCD applications, projects, accounts, and RBAC. Sync, refresh, rollback applications. View sync status, health checks.

gitops kubernetes cd
KeePass-first

ArgoCD is a KeePass-first plugin. ALL connection details are stored in KeePass — the YAML config only controls which environments are enabled and debug settings.

Prerequisites

  • Running ArgoCD server
  • ArgoCD credentials (user/pass or auth token)
  • For RBAC management: kubeconfig with access to ArgoCD namespace

Configuration

Config: ~/.omo/configs/argocd/argocd.yaml

argocd.yaml
environments:
  - name: development
    enabled: true
  - name: production
    enabled: true
  - name: staging
    enabled: true
  - name: sandbox
    enabled: true

debug:
  enabled: false
  log_api_calls: false
  log_responses: false
  request_timeout_seconds: 30

KeePass Secrets

Path: argocd/<environment>/<name>, e.g. argocd/production/prod-cluster

Standard Fields

KeePass FieldMaps ToDescription
TitlenameInstance display name
URLurlArgoCD server URL (e.g. "https://argocd.example.com")
UserNameusernameArgoCD username (for user/pass auth)
PasswordpasswordArgoCD password (for user/pass auth)
NotesdescriptionDescription / notes

Custom Attributes

AttributeDefaultDescription
auth_token—Bearer token (alternative to user/pass)
insecurefalse"true" to skip TLS verification
kubeconfig—Inline kubeconfig YAML (for RBAC management)
kubeconfig_path—Path to kubeconfig file (alternative to inline)
namespaceargocdArgoCD namespace in Kubernetes
tags—Comma-separated tags

Example Tree

KeePass structure
omo (root)
  └── argocd
       ├── development
       │    └── local-argo      (URL=https://localhost:8080, UserName=admin, Password=..., insecure=true)
       └── production
            └── prod-cluster    (URL=https://argocd.prod.com, auth_token=eyJhbG..., namespace=argocd)
Auto-Management

ArgoCD instances are fully managed through KeePass. omo auto-creates placeholder entries with all known custom attributes on first launch, and backfills missing attributes on existing entries.

Troubleshooting

  • No instances found — Create KeePass entries under argocd/<env>/<name>
  • TLS certificate error — Set insecure=true or provide valid certs
  • 401 Unauthorized — Verify auth_token or username/password
  • RBAC features unavailable — Provide kubeconfig or kubeconfig_path
