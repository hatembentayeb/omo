RabbitMQ Plugin

Manage RabbitMQ queues, exchanges, bindings, connections, channels, and nodes. Publish/browse messages, purge queues, monitor cluster health.

messaging broker amqp
Important

RabbitMQ is a "KeePass-first" plugin. ALL connection details are stored in KeePass — the YAML config only controls which environments are enabled and UI settings.

Prerequisites

  • Running RabbitMQ server with Management plugin enabled
  • Network access to AMQP port (default 5672) and Management API port (default 15672)
  • RabbitMQ user credentials

Configuration

Config: ~/.omo/configs/rabbitmq/rabbitmq.yaml

This config does NOT contain connection details. It only controls which environments are enabled:

rabbitmq.yaml
environments:
  - name: development
    enabled: true
  - name: production
    enabled: true
  - name: staging
    enabled: true
  - name: sandbox
    enabled: true

ui:
  refresh_interval: 10
  max_queues_display: 200
  enable_connections: true
  enable_channels: true
  default_view: "overview"

All connection details come from KeePass entries under rabbitmq/<environment>/<name>.

KeePass Secret Schema

Path: rabbitmq/<environment>/<name>, e.g. rabbitmq/production/rabbit-cluster

This plugin uses KeePass as the SOLE source of connection details:

Example tree
omo (root)
  └── rabbitmq
       ├── development
       │    └── local-rabbit    (URL=localhost, UserName=guest, Password=guest, amqp_port=5672, mgmt_port=15672, vhost=/)
       ├── staging
       │    └── staging-rabbit  (URL=staging-rmq.example.com, UserName=app, ...)
       └── production
            └── rabbit-cluster  (URL=10.0.1.50, UserName=admin, Password=..., use_tls=true)
Info

RabbitMQ instances are fully managed through KeePass. To add a new instance, create a new entry in KeePassXC under rabbitmq/<environment>/<name> with the fields above. omo auto-creates placeholder entries on first launch.

Troubleshooting

  • No instances found — Create KeePass entries under rabbitmq/<env>/<name>.
  • Management API unreachable — Ensure Management plugin is enabled and mgmt_port is correct.
  • AMQP connection refused — Check amqp_port and host.
  • Authentication failed — Verify UserName/Password in KeePass.
