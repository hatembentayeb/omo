S3 Plugin

Browse and manage AWS S3 buckets and objects. Preview file contents, upload/download objects, generate presigned URLs.

aws storage cloud

Profiles are loaded from two sources: this config file (with optional KeePass resolution) AND the host machine's ~/.aws/credentials and ~/.aws/config.

Prerequisites

  • AWS account with S3 access
  • AWS credentials (access key + secret key, or IAM role)
  • For S3-compatible: endpoint URL

Configuration

Config: ~/.omo/configs/s3/s3.yaml

This file only controls which environments are enabled and UI settings. All credentials and connection details are stored in KeePass.

~/.omo/configs/s3/s3.yaml
environments:
  - name: default
    enabled: true
  - name: production
    enabled: true

ui:
  refresh_interval: 30
  default_region: "us-east-1"
  max_buckets_display: 100
  max_objects_display: 1000
  enable_metrics: true
  enable_object_browser: true
  enable_policies_viewer: true
  page_size: 50
KeePass-first

All credentials (access key, secret key, region, endpoint, role ARN) are stored exclusively in KeePass entries under s3/<environment>/<name>. The YAML config only controls which environments are enabled and UI behavior. Profiles from ~/.aws/credentials are also merged automatically.

KeePass Secret Schema

Path: s3/<environment>/<name>, e.g. s3/production/storage

KeePass FieldMaps ToDescription

omo (root)
  └── s3

       └── production
            └── minio       (URL=https://minio.internal:9000, UserName=minioadmin, Password=...)
Tip

If you already have ~/.aws/credentials configured, omo will pick up those profiles automatically. KeePass profiles are merged with AWS CLI profiles.

Troubleshooting

  • No buckets listed — Verify credentials and region
  • Access denied — Check IAM permissions
  • Invalid endpoint — Ensure URL is correct for S3-compatible services
  • Timeout — Check network access to AWS/endpoint
