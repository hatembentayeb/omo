PostgreSQL Plugin

Manage PostgreSQL databases, schemas, tables, users, connections, and replication. Execute queries, monitor locks, inspect indexes, view tablespaces.

database sql monitoring

Prerequisites

  • Running PostgreSQL server (9.6+)
  • Network access to PostgreSQL port (default 5432)
  • Database user with appropriate permissions

Configuration

Config: ~/.omo/configs/postgres/postgres.yaml

This file only controls which environments are enabled and UI settings. All connection details are stored in KeePass.

~/.omo/configs/postgres/postgres.yaml
environments:
  - name: development
    enabled: true
  - name: production
    enabled: true

ui:
  refresh_interval: 10
  max_rows_display: 1000
  enable_logs: true
  enable_stats: true
KeePass-first

All connection details (host, port, username, password, database, sslmode) are stored exclusively in KeePass entries under postgres/<environment>/<name>. The YAML config only controls which environments are enabled and UI behavior.

KeePass Secret Schema

Path: postgres/<environment>/<name>, e.g. postgres/production/main-db

KeePass FieldMaps ToDescription
TitlenameInstance display name
URLhostPostgreSQL server hostname or IP
UserNameusernamePostgreSQL username
PasswordpasswordPostgreSQL password
Notes—Description (not used for connection)

No custom attributes — PostgreSQL uses only standard KeePass fields.

Example tree
omo (root)
  └── postgres
       ├── development
       │    └── local     (URL=localhost, UserName=postgres, Password=...)
       └── production
            └── main-db   (URL=pg.prod.internal, UserName=app_user, Password=...)

Views

The PostgreSQL plugin provides 14 views:

  • Users — Database users and roles with privileges.
  • Databases — All databases with size and connection counts.
  • Tables — Tables with row counts, sizes, and schema info.
  • Schemas — Database schemas and their objects.
  • Extensions — Installed PostgreSQL extensions.
  • Connections — Active client connections with state and query info.
  • Stats — Statistics collector data and activity metrics.
  • Config — Server configuration parameters.
  • Logs — PostgreSQL log output when enabled.
  • Locks — Current locks and blocking queries.
  • Indexes — Index definitions and usage statistics.
  • Replication — Replication slots and standby status.
  • Tablespaces — Tablespace locations and sizes.
  • DB Stats — Per-database statistics.

Troubleshooting

  • Connection refused — Check host/port, ensure PostgreSQL is running.
  • Authentication failed — Verify username/password.
  • SSL required — Set sslmode to "require" or "verify-full".
  • Permission denied — User needs appropriate grants.
