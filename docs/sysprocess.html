System Process Plugin

Monitor system processes, CPU, memory, disk usage, and network ports. View the process tree, kill processes, track resource usage with real-time metrics.

system monitoring processes

This plugin requires no configuration — it reads directly from the host OS.

Prerequisites

  • Linux operating system
  • No additional dependencies

Configuration

This plugin has no configuration file. It monitors the local system directly and requires no setup.

Info

The System Process plugin works out of the box with no configuration needed. It reads process information directly from /proc on Linux.

Views

The System Process plugin provides the following views:

  • Processes — List of running processes with PID, name, CPU%, memory%, and user.
  • CPU — Real-time CPU usage metrics.
  • Memory — RAM and swap usage breakdown.
  • Disk — Filesystem usage, mount points, read/write stats.
  • Network Ports — Listening ports and associated processes.
  • Process Tree — Hierarchical parent-child process tree.
  • Process Details — Detailed information about a selected process.
  • Warnings — Resource usage warnings and alerts.

Troubleshooting

  • Permission denied — Some process info requires root or sudo to access.
  • Stale data — Press R to refresh.
  • Cannot kill process — You need appropriate permissions (try running omo with sudo).
  • High CPU from omo — Reduce refresh rate if polling is too aggressive.
