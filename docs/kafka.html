Kafka Plugin

Monitor Kafka brokers, topics, partitions, and consumer groups. Create and delete topics, inspect partition details, track consumer lag, view messages.

streaming messaging monitoring

Prerequisites

  • Running Kafka cluster
  • Network access to bootstrap servers (default port 9092)
  • For SASL: valid credentials
  • For SSL: CA cert, client cert, client key files

Configuration

Config: ~/.omo/configs/kafka/kafka.yaml

This file only controls which environments are enabled and UI settings. All connection details are stored in KeePass.

~/.omo/configs/kafka/kafka.yaml
environments:
  - name: development
    enabled: true
  - name: production
    enabled: true

ui:
  refresh_interval: 10
  max_topics_display: 100
  max_partitions_display: 50
  default_view: "brokers"
  enable_metrics: true
  enable_consumer_groups: true
  enable_message_viewer: true
KeePass-first

All connection details (bootstrap servers, SASL credentials, SSL certificates) are stored exclusively in KeePass entries under kafka/<environment>/<name>. The YAML config only controls which environments are enabled and UI behavior.

KeePass Secret Schema

Path: kafka/<environment>/<name>, e.g. kafka/production/main-cluster

Views

  • Brokers (default) — Cluster broker overview and health
  • Topics — Browse topics with partition counts and replication
  • Partitions — Partition details, leaders, replicas
  • Consumer Groups — Consumer groups, lag, offset tracking
  • Messages — View and browse topic messages

Troubleshooting

  • Connection refused — Check the URL field in KeePass and ensure Kafka brokers are reachable on port 9092.
  • SASL auth failed — Verify username and password in KeePass.
  • SSL handshake error — Verify certificate paths and file permissions for ssl_ca_cert, ssl_cert, and ssl_key.
